Raspberry Pi Mounted Backup Guide

System & Network Mounted Device Backup Procedure

This guide demonstrates a simple method to back up a Raspberry Pi system using a mounted USB drive. A single-line backup command preserves critical configurations, including network settings, Pi-hole data, and other essential system files, allowing easy restoration or migration to a new SD card. 

#!/bin/bash

# Plug in and mount USB/SSD
lsblk    # confirm the mount point

# Create new folder for backup called pihole-backup
sudo mkdir -p /mnt/usb/pihole-backup

# Create comprehensive backup archive for all config
sudo tar -czvf /mnt/usb/pihole-backup/pihole-full-backup-$(date +%F).tar.gz \
    /etc/pihole \
    /etc/dhcpcd.conf \
    /etc/network \
    /etc/ssh \
    /etc/ufw \
    /etc/fail2ban

# Verify backup
ls -lh /mnt/usb/pihole-backup
tar -tzvf /mnt/usb/pihole-backup/pihole-full-backup-$(date +%F).tar.gz | head

# Unmount USB
sudo umount /mnt/usb
