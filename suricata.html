Network Security Operations Dashboard

Version 1.0 March 2026
Private deployment — homelab network security monitoring infrastructure.

Overview

After deploying Suricata IDS on a dedicated Raspberry Pi with SPAN port mirroring to monitor all network traffic, I built a comprehensive security operations dashboard to surface actionable intelligence from raw IDS data. The system ingests Suricata's eve.json event stream through a multi-tier aggregation pipeline and generates a self-hosted web dashboard providing real-time visibility into network behavior, threat patterns, and device activity across 47+ connected devices.

The dashboard unifies data from two independent systems, Suricata IDS and Pi-hole DNS, into a single operational view, enabling threat investigation without context switching. Each IP is clickable, opening a dual-pane modal combining DNS query history alongside network traffic connections, protocols, and ports in a single investigative interface.

Bash SQLite3 JSON / jq HTML / CSS / JavaScript Nginx PKI / OpenSSL Leaflet.js Suricata IDS Pi-hole API

Cross-System DNS Telemetry

The dashboard reaches beyond Suricata's own event stream, making live API calls to a separate Pi-hole DNS server to pull DNS summary stats, block rates, top queried domains, and top clients directly into the security operations view.

Network traffic and DNS telemetry from two independent systems, unified with no manual correlation required. 122,000+ DNS queries tracked daily with a 34.7% block rate across 258,000+ domains on the blocklist.

  • Live Pi-hole API v6 calls refreshed every 3 minutes
  • Top clients and top domains surfaced alongside IDS traffic data
  • Enables DNS-level investigation without leaving the dashboard

Dual-Map Architecture

A dual-map architecture separates operational (24hr) and historical views. Circle markers are sized proportionally by connection volume and color-coded by threat level: trusted, neutral, or high-risk, providing immediate visual triage of external traffic patterns.

Clicking any country opens a full geo detail page with per-IP breakdowns including city-level location, protocol and port data, connection counts, and first and last seen timestamps. All geolocation is performed locally via a DB-IP database with no external API calls and no rate limiting.

  • 5,237 IPs cached in local geo database
  • Trusted, neutral, and high-risk country classification
  • Filterable by country and threat level

Unified Per-IP Investigation

Every IP in the dashboard is clickable. Selecting one opens a dual-pane modal pulling from both data sources simultaneously. Pi-hole DNS query history appears on the left and Suricata network traffic on the right, enabling full context on any device or external connection without switching tools.

  • DNS pane shows queried domains, record types, allow/block status, and timestamps
  • Network pane shows outgoing connections, protocols, ports, and last-seen times
  • Clickable external IPs link to WHOIS, AbuseIPDB, VirusTotal, and geolocation lookups
  • Time-aware display across Last Hour, Last 24h, and Last 7 Days

Multi-Tier Aggregation Pipeline

Three independent aggregation scripts process Suricata's eve.json at different intervals. Hourly runs every 5 minutes, daily every 15 minutes, and weekly every hour. Each produces structured JSON consumed downstream by the dashboard, enabling time-aware traffic analysis without re-parsing raw logs on every load.

  • Top sources and destinations per timeframe with packet counts
  • Device name mapping applied consistently across all timeframes
  • Identifies behavioral changes over time, separating hourly spikes from sustained patterns
  • Weekly aggregator handles eve.json rotation across multiple log files

Continuous Device Inventory

The dashboard continuously monitors Suricata's event stream for IP addresses not present in the known-devices registry. New devices are flagged immediately with first-seen and last-seen timestamps, packet counts, and an inline name-assignment workflow, maintaining a complete and auditable device inventory without manual network scanning.

  • 46 devices tracked across the network
  • New, returning, and time-filtered views across Last 24h, 7d, and 30d
  • Full device history audit trail persisted to JSON
  • Name assignments propagate across all dashboard sections

Infrastructure and Security

  • PKI-Secured Deployment: Self-hosted on Nginx with custom CA and CSR-issued certificates, HTTPS enforcement, and Content Security Policy headers, matching the security posture of a production internal tool
  • Defensive Programming: All aggregator scripts implement graceful failure handling, exit code validation, and automated alerting on pipeline anomalies. Failures surface loudly, never silently
  • Local-Only Geolocation: DB-IP city database queried via mmdblookup with 5,237 IPs cached and zero redundant lookups once warmed, with no external dependencies
  • USB Log Storage: All Suricata logs written to USB-mounted storage to prevent SD card wear, with 90-day retention and daily logrotate rotation
  • Static Dashboard: No user input processing and read-only display only, eliminating injection attack surface entirely
