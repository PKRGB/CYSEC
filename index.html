Home Network Overhaul & Segmentation

I redesigned my home network to improve security and performance, including isolating IoT devices and monitoring traffic...

  • Set up VLANs for segmentation
  • Moved IoT to isolated VLAN
  • Used pfSense for firewall
  • Analyzed traffic with Wireshark/nmap

This is a sanitized overview of my segmented home network setup. The ISP modem is bridged, and all routing, DHCP, and DNS control is handled by a custom router with firewall rules. Instead of allowing clients to choose their own DNS, all DNS queries are routed through Pi-hole at the firewall level, ensuring consistent blocking and visibility across the entire network. The network is segmented into:

  • VLAN 10: Primary/trusted devices
  • VLAN 20: IoT devices, isolated from the trusted devices subnet
Logs from Pi-hole processing and traffic from all devices are exported to a local SIEM system (Splunk) to support alerting, analysis, and automation experiments.

Tools: pfSense, Wireshark, nmap, VLAN, SPLUNK

Scripts

Python

I utilized Python to create a simple Caesar Cipher which allows the user to input their plaintext password, a "key" number of their choice and then outputs ciphertext. The program incorporates the option to decrypt with the correct menu selection, via input of the ciphertext and "key". While this is not a truly secure method of data encryption, it does offer a layer of protection against shoulder-surfing or casual interception.

Java

I created a password strength checker in Java, which users can utilize to gain feedback on the strength of a potential password. If the password is deemed to be weak or only moderately strong, the output will provide suggestions for strengthening; incorporating capitalization, special characters, numbers and minimum length. The script will also randomly generate these variables and add them to the user-provided input, to provide a stronger 'suggested' password built upon the input the user initially provided.
