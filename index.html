Home Network Overhaul & Segmentation

Below is a sanitized overview of my home network topology, with a focus on security monitoring and threat detection. The architecture implements defense-in-depth principles through network segmentation, passive IDS monitoring, honeypot deployment, and centralized logging.

In my updated network architecture, the ISP modem provides connectivity to both production and honeypot networks via separate ports, with an isolated router supporting the honeypot. In the production network, all routing, DHCP, and DNS control is handled by a custom router with strict firewall rules to filter inbound and outbound traffic. DNS queries are forced through Pi-hole with Unbound for recursive resolution at the firewall level, ensuring consistent ad-blocking and eliminating reliance on third-party DNS providers.

My updated network architecture includes:

  • VLAN 20: Primary/trusted devices (workstations, laptops, mobile devices)
  • VLAN 10: IoT devices (smart home, cameras, sensors) - isolated from trusted network to prevent lateral movement
  • IDS Monitoring: Suricata passive IDS deployed with SPAN port mirroring for complete traffic visibility without inline performance impact
  • DNS Security: Pi-hole + Unbound for ad-blocking and fully recursive DNS resolution
  • Honeypot Network: Isolated Cowrie SSH/Telnet honeypot via isolated router on different subnet than production network, for threat intelligence and attack pattern analysis
  • Centralized Logging: All security events aggregated to external storage for correlation and forensic analysis

Security Features: VLAN segmentation, passive IDS monitoring, SSH hardening (key-based auth, fail2ban), strict firewall rules, honeypot deployment, zero-trust architecture

Tools & Technologies: Suricata IDS, Pi-hole, Unbound DNS, Cowrie Honeypot, pfSense/custom router, VLAN (802.1Q), Wireshark, nmap, tcpdump, SPAN/port mirroring, Linux (Debian/Raspberry Pi OS)

Pi-hole + Unbound: DNS Security & Privacy

To eliminate ad-serving, block telemetry, and achieve complete DNS privacy, I deployed Pi-hole with Unbound for fully recursive DNS resolution. This configuration ensures all DNS queries are resolved directly from root nameservers without relying on third-party DNS providers like Google or Cloudflare, eliminating a common privacy and security risk.

  • Installed and configured Pi-hole with Unbound as recursive DNS resolver
  • Configured router to force all DNS traffic through Pi-hole at the firewall level (no client bypass)
  • Set up custom blocklists and whitelist exceptions for legitimate services
  • Analyzed DNS query logs to identify suspicious domains and IOC patterns
  • Configured DNSSEC validation for query integrity and authenticity
  • Monitored DNS performance metrics and cache hit rates for optimization
  • Integrated DNS logs with centralized logging for security correlation
