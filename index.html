Home Network Overhaul & Segmentation

Below is a sanitized overview of my home network topology, with a focus on security monitoring and threat detection. The architecture implements defense-in-depth principles through network segmentation, passive IDS monitoring, honeypot deployment, and centralized logging.

In my updated network architecture we utilize a managed switch, with port isolation, to an isolated router supporting the honeypot for ongoing threat analysis and education. In the production network, all routing, DHCP, and DNS control is handled by a custom router with strict firewall rules governing inbound and outbound traffic. DNS traffic is forcibly redirected to an internal DNS stack at the firewall level to ensure consistent enforcement of filtering and privacy controls across the network.

My updated network architecture includes:

  • VLAN 20: Primary/trusted devices (workstations, laptops, mobile devices)
  • VLAN 10: IoT devices (smart home, cameras, sensors) - isolated from trusted network to prevent potential lateral movement in the event a device is compromised
  • IDS Monitoring: Suricata passive IDS deployed with SPAN port mirroring for complete LAN traffic visibility without inline performance impact, and implemented real-time security alerting via custom bash script that monitors Suricata IDS logs, updating a dashboard automatically at specific intervals and triggers desktop notifications on Pi's 7-inch display for immediate threat visibility
  • DNS Security: Pi-hole + Unbound for ad-blocking and telemetry-reduction via fully recursive DNS resolution, as well as the implementation of DNSSEC
  • Honeypot Network: Isolated Cowrie SSH/Telnet honeypot via separate router on different subnet than production network, port isolated virtually with managed switch, for threat intelligence and attack pattern analysis
  • Centralized Logging: All security events aggregated to external storage for correlation and forensic analysis

Security Features: VLAN segmentation, passive IDS monitoring, SSH hardening (key-based auth, fail2ban), strict firewall rules, honeypot deployment, zero-trust architecture

Tools & Technologies: Suricata IDS, Pi-hole, Unbound DNS, Cowrie Honeypot, pfSense/custom router, VLAN (802.1Q), Wireshark, nmap, tcpdump, SPAN/port mirroring, Linux (Debian/Raspberry Pi OS)

Pi-hole + Unbound: DNS Security & Privacy

To eliminate ad-serving, block telemetry, and achieve DNS privacy, I deployed Pi-hole with Unbound for fully recursive DNS resolution. This configuration ensures all DNS queries are resolved directly from root and authoritative nameservers rather than forwarded to public resolvers, eliminating reliance on third-party DNS resolvers like Google or Cloudflare.

Approaching DNS resolution in this manner eliminates a common availability, privacy and security risk by not relying on a singular upstream DNS resolver, circumventing potential DNS/connectivity issues in the event of a third-party server's unexpected downtime, and also reducing fingerprinting risks associated with centralized query logging. DNSSEC validation is enabled to provide cryptographic assurance of DNS authenticity; which mitigates DNS spoofing, cache poisoning, and Man-in-the-Middle attacks, upholding the Confidentiality and Integrity pillars of the CIA triad.

Implementation Process:

  • Installed and configured Pi-hole with Unbound as recursive DNS resolver
  • Configured router to force all DNS traffic through Pi-hole at the firewall level (no client bypass)
  • Set up custom blocklists and whitelist exceptions for legitimate services
  • Analyzed DNS query logs to identify suspicious domains and IOC patterns
  • Configured DNSSEC validation for query integrity and authenticity
  • Monitored DNS performance metrics and cache hit rates for optimization
  • Integrated DNS logs with centralized logging for security correlation

Tools & Technologies: Pi-hole, Unbound DNS, Raspberry Pi OS, SSH, DNSSEC, DNS query logging, sqlite3, regex filtering

Key Achievement: Achieved complete DNS privacy and ad-blocking across entire network with 100% query coverage ensuring no device can bypass filtering, even with hardcoded DNS settings.


Custom DNS Query Analytics Dashboard

After deploying Pi-hole with Unbound as my recursive DNS server, I built an automated bash script to extend visibility of historical queries and other stats beyond the metrics provided by the standard Pihole GUI admin interface, in order to provide anomaly detection and deeper insights. This tool analyzes 90+ days of DNS query logs (8M+ queries) and generates interactive HTML reports with queries by hour of day, day of the week, query total, as well as client health scoring and usage pattern analysis.

Go to Interactive Dashboard

The data can be sorted by IP, Block Rate %, Device Name or Health Score but defaults ranking from lowest Health Score to highest, spotlighting devices which need the most attention. The system incorporates arp-scan for network discovery and automatically attempts to identify device names via hostname resolution and MAC address vendor lookup, when available.

Script Design & Functional Capabilities:

  • Iterative Improvement: Continuously refined based on real-world network insights to improve performance, usability, and aesthetics while incorporating new security metrics
  • Device Identification & Attribution: Leveraged arp-scan within a Bash pipeline to correlate static IP addresses and MAC address data to enrich the dashboard by automatically identifying device manufacturer metadata, in order to facilitate device identification to streamline investigations
  • Health Scoring Algorithm: Composite security metric evaluating block rates, NXDOMAIN failures, query volume anomalies, and domain diversity to flag potentially compromised or misconfigured devices
  • Responsive Interface: Accordion-based UI with sortable columns, mobile optimization, and UTF-8 encoding for cross-platform compatibility
  • Automation & Security: Non-anonymized version runs via cron for weekly DNS pattern analysis, to identify anomalies and trends

Technologies: Bash, SQLite3, HTML/CSS/JavaScript, SQL optimization
